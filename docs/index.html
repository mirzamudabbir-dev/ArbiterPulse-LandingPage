How Pulse Works

Pulse runs a lightweight daemon on your machine (under 3% CPU) that watches four signals simultaneously:

1. Process & port fingerprinting

Identifies AI tools by their process names and network ports. Ollama on port 11434. Cursor by binary path. Claude Code by process name.

2. Network traffic patterns

Monitors connections to known AI provider endpoints — Anthropic, OpenAI, Google, Cursor, GitHub Copilot — using metadata only. Never payload content.

3. Hardware behavioral signatures

Reads GPU utilisation, power draw, and thermal state directly from Apple Silicon hardware (IOReport). Distinguishes inference workloads from rendering or gaming by their hardware fingerprint.

4. OS event tracing

Subscribes to process lifecycle events to detect workload starts and stops with sub-second latency.

These four signals are combined into a confidence score for each detected workload. The analytics engine then correlates workloads with hardware behaviour every five seconds, surfacing patterns — memory saturation, GPU contention, thermal attribution — as plain-language insights.

Everything runs locally. Nothing leaves your machine.
